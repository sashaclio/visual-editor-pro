# Contributing to Visual Editor Pro

Thank you for your interest in contributing to Visual Editor Pro.

This project is intended to grow into a maintainable open-source workflow for structured photo diagnosis, retouching recommendations, and editorial-style visual direction. Contributions are welcome across prompt design, documentation, examples, schema design, testing, and tooling.

## Ways to Contribute

Contributions are welcome in the following areas:

- Improve the skill wording for clarity, precision, and consistency
- Add or refine example inputs and outputs
- Propose structured output schemas for diagnosis and parameter recommendations
- Improve formatting for Lightroom and Capture One outputs
- Suggest evaluation methods or benchmark datasets
- Add tooling such as CLI wrappers, validation scripts, or lightweight demos
- Report ambiguous cases, edge cases, or failure patterns
- Improve documentation and repository organization

## Contribution Guidelines

Please follow these principles when contributing:

- Keep outputs actionable rather than vague or overly stylistic
- Preserve professional terminology such as Tone Curve, HSL, Clarity, and Capture One parameter naming
- Favor structured workflow design over one-off prompt tricks
- Keep editorial references restrained and credible
- When adding examples, prefer realistic photography scenarios over synthetic filler content
- If changing the skill behavior, explain why the new behavior improves reliability, usability, or maintainability

## Suggested Workflow

1. Fork the repository
2. Create a branch for your change
3. Make focused edits
4. Update documentation or examples when relevant
5. Open a pull request with a clear explanation of the change

## Pull Request Notes

A good pull request should include:

- What changed
- Why it changed
- What part of the workflow it affects
- Whether the change affects outputs, formatting, or compatibility
- Example before/after output when applicable

## Issues

When opening an issue, please include as much context as possible:

- The type of image or workflow scenario
- The current output behavior
- Why the behavior is incorrect, incomplete, or unclear
- A suggested improvement if you have one

## Scope

This repository is currently focused on:

- Structured photo diagnosis
- Lightroom recommendations
- Capture One recommendations
- Editorial-style image direction
- Skill design for maintainable AI workflows

Broader productization ideas such as full apps, hosted services, or commercial integrations may be discussed, but the repository should remain useful as a clean open-source foundation.

## Code of Collaboration

Be constructive, specific, and respectful. The goal is to make the workflow more useful, more maintainable, and easier for others to build on.
